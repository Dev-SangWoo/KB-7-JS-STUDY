# 5주차 발표

## 1. 클로저란?

클로저는 **외부 함수의 실행이 끝난 뒤에도, 내부 함수가 외부 함수의 변수를 계속 참조할 수 있는 현상**이다.

쉽게 말하면,

> 외부 함수는 이미 종료되었지만, 내부 함수가 외부 함수의 변수를 기억하고 있는 상태
> 

라고 볼 수 있다.

정확히 정리하면 다음과 같다.

> 클로저란 어떤 함수 A에서 선언한 변수 `a`를 참조하는 내부 함수 B가 외부로 전달될 경우, A의 실행 컨텍스트가 종료된 이후에도 변수 `a`가 사라지지 않는 현상이다.
> 

---

## 2. 실행 컨텍스트와 클로저의 관계

클로저를 이해하려면 2장에서 배운 **실행 컨텍스트**와 연결해서 봐야 한다.

함수가 실행되면 실행 컨텍스트가 생성된다.

실행 컨텍스트 안에는 `LexicalEnvironment`가 있고, 그 안에는 크게 두 가지가 있다.

| 구성 요소 | 역할 |
| --- | --- |
| `environmentRecord` | 현재 함수 내부의 변수, 매개변수, 함수 선언 등을 저장 |
| `outerEnvironmentReference` | 현재 함수가 선언된 바깥 환경을 참조 |

내부 함수는 자신의 `environmentRecord`에서 변수를 먼저 찾는다.

만약 찾지 못하면 `outerEnvironmentReference`를 따라 바깥 함수의 `LexicalEnvironment`에서 변수를 찾는다.

이 과정이 바로 **스코프 체인**이다.

---

## 3. 기본 예제: 외부 함수의 변수를 참조하는 내부 함수

```
varouter=function () {
vara=1;

varinner=function () {
console.log(++a);
  };

inner();
};

outer();
```

### 실행 흐름

1. `outer()`가 실행된다.
2. `outer` 함수의 실행 컨텍스트가 생성된다.
3. `outer` 내부에서 변수 `a`가 선언되고 값 `1`이 저장된다.
4. `inner()`가 실행된다.
5. `inner` 함수 안에는 `a`가 없으므로 바깥 함수인 `outer`의 `LexicalEnvironment`에서 `a`를 찾는다.
6. `++a`가 실행되어 `2`가 출력된다.
7. `inner` 실행이 끝난다.
8. `outer` 실행도 끝난다.

이 경우에는 `inner`가 `outer` 내부에서 바로 실행되고 끝난다.

따라서 `outer` 함수가 종료된 후에는 `a`를 더 이상 참조할 함수가 없다.

그래서 이 예제는 클로저의 핵심 상황은 아니다.

---

## 4. `return inner()`와 `return inner`의 차이

클로저를 이해할 때 가장 중요한 차이는 이것이다.

```
returninner();
```

와

```
returninner;
```

는 완전히 다르다.

---

## 4-1. `return inner()`인 경우

```
varouter=function () {
vara=1;

varinner=function () {
return++a;
  };

returninner();
};

varouter2=outer();

console.log(outer2);// 2
```

### 설명

`return inner()`는 `inner` 함수를 실행한 결과를 반환한다.

즉, 함수 자체를 반환하는 것이 아니라 숫자 `2`를 반환하는 것이다.

따라서 `outer()`가 끝난 뒤에는 `inner` 함수를 다시 호출할 수 없다.

`a`를 계속 참조할 대상도 없다.

그래서 `outer` 함수의 실행이 끝나면 `a`는 더 이상 유지될 필요가 없다.

---

## 4-2. `return inner`인 경우

```
varouter=function () {
vara=1;

varinner=function () {
return++a;
  };

returninner;
};

varouter2=outer();

console.log(outer2());// 2
console.log(outer2());// 3
```

### 설명

여기서는 `inner()`를 실행한 결과가 아니라 `inner` 함수 자체를 반환한다.

```
varouter2=outer();
```

이 코드가 실행되면 `outer2`에는 `inner` 함수가 저장된다.

즉, `outer` 함수는 이미 종료되었지만, `outer2()`를 통해 `inner` 함수를 계속 실행할 수 있다.

`inner` 함수는 실행될 때마다 외부 함수 `outer`의 변수 `a`를 참조한다.

그래서 `a`는 사라지지 않고 계속 유지된다.

출력 결과는 다음과 같다.

```
console.log(outer2());// 2
console.log(outer2());// 3
```

첫 번째 호출에서는 `a`가 `1`에서 `2`가 된다.

두 번째 호출에서는 이전 값이 유지되어 `2`에서 `3`이 된다.

이것이 클로저다.

---

## 5. 왜 outer 함수가 끝났는데도 a가 사라지지 않을까?

일반적으로 함수 실행이 끝나면 실행 컨텍스트는 콜 스택에서 제거된다.

그러면 그 함수 안의 지역변수도 더 이상 사용할 수 없게 된다.

하지만 클로저 상황에서는 다르다.

```
returninner;
```

이렇게 내부 함수가 외부로 전달되면, 내부 함수는 나중에 다시 실행될 수 있다.

그리고 내부 함수 `inner`는 외부 함수 `outer`의 변수 `a`를 참조하고 있다.

따라서 자바스크립트 엔진은 `a`가 들어 있는 `outer`의 `LexicalEnvironment`를 바로 제거하지 않는다.

이유는 간단하다.

> 아직 `inner` 함수가 `a`를 사용할 가능성이 있기 때문이다.
> 

가비지 컬렉터는 어떤 값을 참조하는 대상이 하나라도 있으면 그 값을 메모리에서 제거하지 않는다.

그래서 `outer` 함수의 실행 컨텍스트가 종료되어 콜 스택에서 빠져나간 뒤에도, `inner`가 참조하는 변수 `a`는 계속 유지된다.

## 6. 클로저 발생 조건

클로저가 발생하려면 다음 조건이 필요하다.

1. 외부 함수가 있다.
2. 외부 함수 안에 내부 함수가 있다.
3. 내부 함수가 외부 함수의 변수를 참조한다.
4. 내부 함수가 외부로 전달된다.
5. 외부 함수가 종료된 뒤에도 내부 함수가 외부 함수의 변수를 계속 참조한다.

즉, 단순히 내부 함수가 있다고 해서 무조건 클로저가 되는 것은 아니다.

핵심은 다음이다.

> 내부 함수가 외부 함수의 변수를 참조하고, 그 내부 함수가 외부로 전달되어 외부 함수 종료 후에도 변수가 유지되는가?
> 

---

## 7. return만 클로저가 아니다

클로저는 꼭 `return`으로 내부 함수를 반환할 때만 발생하는 것이 아니다.

내부 함수를 외부로 전달하는 방식에는 여러 가지가 있다.

대표적으로 다음과 같은 경우가 있다.

```
setTimeout(callback,time);
setInterval(callback,time);
addEventListener('click',callback);
```

예를 들어 다음 코드를 보자.

```
(function () {
varcount=0;

button.addEventListener('click',function () {
console.log(++count);
  });
})();
```

즉시 실행 함수는 이미 종료된다.

하지만 버튼을 클릭할 때마다 `count` 값은 계속 증가한다.

이유는 `addEventListener`에 전달된 콜백 함수가 `count`를 계속 참조하고 있기 때문이다.

즉, 내부 함수가 외부 객체인 DOM 이벤트에 콜백으로 전달되었고, 그 콜백 함수가 외부 함수의 지역변수 `count`를 기억하고 있으므로 이것도 클로저다.

---

### setInterval / setTimeout 예제

```
(function () {
vara=0;
varintervalId=null;

varinner=function () {
if (++a>=10) {
clearInterval(intervalId);
    }

console.log(a);
  };

intervalId=setInterval(inner,1000);
})();
```

이 코드는 즉시 실행 함수이다.

```
(function () {
// 코드
})();
```

즉시 실행 함수는 한 번 실행되고 바로 종료된다.

따라서 원래라면 함수 안의 지역변수인 `a`와 `intervalId`는 사라져야 한다.

하지만 이 코드에서는 `inner` 함수가 `setInterval`에 콜백 함수로 전달된다.

```
intervalId=setInterval(inner,1000);
```

`setInterval`은 브라우저의 `window` 객체가 관리하는 외부 메서드이다.

즉, `inner` 함수가 즉시 실행 함수 바깥으로 전달된 것이다.

그런데 `inner` 함수 안에서는 외부 함수의 지역변수인 `a`와 `intervalId`를 사용하고 있다.

```
if (++a>=10) {
clearInterval(intervalId);
}
```

그래서 즉시 실행 함수가 끝난 뒤에도 `inner` 함수가 계속 `a`와 `intervalId`를 참조할 수 있어야 한다.

따라서 `a`와 `intervalId`는 가비지 컬렉터의 수집 대상이 되지 않고 메모리에 남는다.

이것이 클로저이다.

---

### (2) eventListener 예제

```
(function () {
varcount=0;
varbutton=document.createElement('button');

button.innerText='click';

button.addEventListener('click',function () {
console.log(++count,'times clicked');
  });

document.body.appendChild(button);
})();
```

이 코드도 즉시 실행 함수이다.

즉시 실행 함수는 실행이 끝나면 종료된다.

하지만 여기서는 버튼에 클릭 이벤트를 등록하고 있다.

```
button.addEventListener('click',function () {
console.log(++count,'times clicked');
});
```

`addEventListener`에 전달된 함수는 클릭 이벤트가 발생했을 때 나중에 실행된다.

즉, 이 함수도 외부 객체인 DOM에 콜백 함수로 전달된 것이다.

그런데 이 콜백 함수 안에서는 즉시 실행 함수의 지역변수인 `count`를 사용하고 있다.

```
console.log(++count,'times clicked');
```

즉시 실행 함수는 이미 끝났지만, 버튼을 클릭할 때마다 `count`가 증가한다.

이 말은 `count`가 사라지지 않고 계속 메모리에 남아 있다는 뜻이다.

왜냐하면 클릭 이벤트에 등록된 콜백 함수가 `count`를 계속 참조하고 있기 때문이다.

그래서 이것도 클로저이다.

---

## 기억하면 좋은 거

| 코드 | 의미 | 클로저 핵심 여부 |
| --- | --- | --- |
| `return inner()` | 내부 함수를 실행한 결과 반환 | 핵심 아님 |
| `return inner` | 내부 함수 자체 반환 | 클로저 발생 |
