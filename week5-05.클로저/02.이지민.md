# 클로저 (Closure)

## 클로저란?

> 내부함수가 외부함수의 변수를 참조할 때 발생하며, 외부함수가 종료돼도 참조한 변수는 메모리에서 사라지지 않음

## 메모리 관리

클로저는 외부함수의 변수를 의도적으로 메모리에 유지시킴.
**다 쓴 시점에 null을 할당해서 참조를 끊어줘야 GC(가비지컬렉터)가 수거해서 메모리 해제**

### 1. return 방식

```js
var outer = (function () {
  var a = 1;
  var inner = function () {
    return ++a;
  };
  return inner;
})();

console.log(outer()); // 2
console.log(outer()); // 3
outer = null; // 참조 끊기 → GC 수거
```

### 2. setInterval 방식

```js
(function () {
  var a = 0;
  var intervalId = null;
  var inner = function () {
    if (++a >= 10) {
      clearInterval(intervalId); // 타이머 중지
      inner = null; // 참조 끊기 → GC 수거
    }
    console.log(a);
  };
  intervalId = setInterval(inner, 1000);
})();
```

### 3. eventListener 방식

```js
(function () {
  var count = 0;
  var button = document.createElement('button');
  button.innerText = 'click';

  var clickHandler = function () {
    console.log(++count, 'times clicked');
    if (count >= 10) {
      button.removeEventListener('click', clickHandler);
      clickHandler = null; // 참조 끊기 → GC 수거
    }
  };
  button.addEventListener('click', clickHandler);
  document.body.appendChild(button);
})();
```

| 방식               | null 처리 대상              | 해제 방법                                                            |
| ------------------ | --------------------------- | -------------------------------------------------------------------- |
| return 방식        | 반환된 함수를 담은 변수     | `outer = null`                                                       |
| setInterval 방식   | 콜백 함수 + 타이머 중지     | `inner = null` + `clearInterval(intervalId)`                         |
| eventListener 방식 | 이벤트 핸들러 + 이벤트 제거 | `clickHandler = null` + `removeEventListener('click', clickHandler)` |

> **공통 원칙 = 클로저가 참조하고 있는 것들을 null로 끊어주면 GC가 수거해서 메모리 해제**

## 콜백 함수에서 외부변수 참조하는 방법 3가지

**5-6 방식 - 클로저 직접 사용 (단순할 때)**

```js
fruits.forEach(function (fruit) {
  // fruit = 'apple', 'banana', 'peach' 순서로 실행
  var $li = document.createElement('li');
  $li.innerText = fruit; // li 텍스트에 과일 이름 설정
  $li.addEventListener('click', function () {
    alert('your choice is ' + fruit); // 클릭 시 forEach의 fruit를 클로저로 기억해서 출력
  });
  $ul.appendChild($li);
});
```

- forEach가 실행될 때마다 각 fruit를 클로저로 기억
- apple li 클릭 → 'your choice is apple' / banana li 클릭 → 'your choice is banana'
- 간단하고 직관적, 재사용 필요 없을 때 적합

---

**5-8 방식 - bind 사용 (재사용 필요할 때)**

```js
var alertFruit = function (fruit) {
  // 외부로 꺼낸 함수 → 재사용 가능
  alert('your choice is ' + fruit);
};
fruits.forEach(function (fruit) {
  $li.addEventListener('click', alertFruit.bind(null, fruit));
  // bind로 fruit를 미리 고정 → 클릭 시 이벤트 객체 대신 fruit가 들어감
  // null = this 신경 안 씀
});
```

- `alertFruit`를 외부로 꺼내서 다른 곳에서도 직접 호출 가능
- bind 없이 `alertFruit`만 넘기면 addEventListener가 fruit 자리에 이벤트 객체를 넣어버림
- 단, bind 사용 시 this가 바뀌는 문제 있음

---

**5-9 방식 - 고차함수 사용 (5-8의 문제 해결)**

```js
var alertFruitBuilder = function (fruit) {
  // fruit를 인자로 받아서
  return function () {
    // 함수를 반환
    alert('your choice is ' + fruit); // 반환된 함수가 fruit를 클로저로 기억
  };
};
fruits.forEach(function (fruit) {
  $li.addEventListener('click', alertFruitBuilder(fruit));
  // alertFruitBuilder(fruit) 실행 → 내부 함수 반환 → 그 함수가 콜백으로 등록
});
```

- `alertFruitBuilder('apple')` 실행하면 apple을 기억하는 함수가 반환됨
- 반환된 함수가 addEventListener의 콜백으로 등록
- 클로저로 fruit 기억 + this 문제 없음 → 5-8보다 상위호환

> **단순하면 5-6, 재사용 필요하면 5-9**
