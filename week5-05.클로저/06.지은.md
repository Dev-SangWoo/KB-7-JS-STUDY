# 📘 5장: 클로저 (Closure) 정리

## 1. 클로저의 정의 및 원리

- **정의**
    
    외부 함수 A에서 선언한 변수 a를 참조하는 내부 함수 B를 외부로 전달할 경우,
    
    A의 실행 컨텍스트가 종료된 후에도 변수 a가 메모리에서 사라지지 않는 현상이다.
    
- **원리**
    
    가비지 컬렉터(GC)는 어떤 값을 참조하는 변수가 하나라도 있다면 수집 대상에서 제외한다.
    
    내부 함수가 외부 변수를 참조하고 있으면, 외부 함수의 실행이 끝나도 해당 변수의 **Lexical Environment**는 소멸하지 않고 유지된다.
    

---

## 2. 클로저와 메모리 관리

- **메모리 누수인가?**
    
    메모리 소모는 클로저의 본질적인 특성일 뿐이며, 관리법만 잘 파악하면
    
    '메모리 누수'가 아닌 의도된 설계로 활용할 수 있다.
    
- **메모리 해제**
    
    클로저의 필요성이 사라진 시점에 식별자에 참조형이 아닌 기본형 데이터(`null` 또는 `undefined`)를 할당하여 참조 카운트를 0으로 만들면 GC가 수거해간다.
    

---

## 3. 주요 활용 사례

클로저는 실무에서 다음과 같은 목적으로 광범위하게 활용된다.

| 활용 사례 | 설명 |
| --- | --- |
| 콜백 함수 내부 데이터 사용 | 콜백 함수를 고차 함수로 구현하여 외부 인자를 클로저로 기억시키고 나중에 활용한다. |
| 접근 권한 제어 (정보 은닉) | 외부로 노출할 멤버는 `return`하고, 나머지는 지역 변수로 두어 `public`과 `private` 멤버를 구분한다. |
| 부분 적용 함수 | 인자 중 일부를 미리 넘겨두어 기억시켰다가 나중에 나머지 인자를 받아 실행한다. 대표적인 예로 **디바운스(Debounce)**가 있다. |
| 커링 함수 | 여러 개의 인자를 받는 함수를 하나의 인자만 받는 함수들로 나눠 체인 형태로 구성한다. Redux 미들웨어 등에서 자주 사용된다. |
