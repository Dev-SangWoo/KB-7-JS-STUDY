---

## 1. 콜백 함수에서의 this

### 1.1 개념

콜백 함수는 다른 함수(또는 메서드)에 의해 실행되는 함수이다.

이때 콜백 함수 내부에서의 `this`는 **콜백을 호출한 함수(제어권을 가진 함수)**가 결정한다.

---

### 1.2 기본 규칙

- 별도로 `this`를 지정하지 않으면
→ 일반 함수 호출 방식과 동일하게 동작
→ `this`는 전역 객체를 가리킨다

---

### 1.3 예제 분석

### (1) setTimeout

```
setTimeout(function () {
console.log(this);
},300);
```

- 콜백 실행 주체: `setTimeout`
- this 지정 여부: 없음
- 결과: 전역 객체

---

### (2) forEach

```
[1,2,3].forEach(function (x) {
console.log(this,x);
});
```

- 콜백 실행 주체: `forEach`
- this 지정 여부: 없음
- 결과: 전역 객체

---

### (3) addEventListener

```
document.querySelector('#btn')
.addEventListener('click',function () {
console.log(this);
  });
```

- 콜백 실행 주체: `addEventListener`
- 특징: 콜백 실행 시 `this`를 명시적으로 지정
- 결과: 이벤트가 발생한 HTML 요소

---

### 1.4 핵심 정리

- 콜백 함수의 `this`는 고정되지 않는다
- 호출하는 함수가 결정한다

정리:

```
this = 제어권을 가진 함수가 정의한 값
```

## 2. 생성자 함수에서의 this

### 2.1 개념

생성자 함수는 `new` 키워드와 함께 호출되어

객체(인스턴스)를 생성하는 함수이다.

이때 함수 내부의 `this`는

**새로 생성된 인스턴스를 가리킨다**

---

### 2.2 예제

```
varCat=function (name,age) {
this.name=name;
this.age=age;
};

varchoco=newCat('초코',7);
```

---

### 2.3 동작 과정

`new Cat('초코', 7)` 실행 시:

1. 빈 객체 생성
2. `this`를 해당 객체로 설정
3. 프로퍼티 할당 (`this.name`, `this.age`)
4. 객체 반환

---

### 2.4 결과

```
choco= {
  name:'초코',
  age:7
}
```

→ `this === choco`

---

### 2.5 new 없이 호출한 경우

```
varchoco=Cat('초코',7);
```

- 일반 함수 호출로 동작
- `this` → 전역 객체
- 객체 생성되지 않음

---

### 2.6 핵심 정리

```
new 사용 → this = 새 인스턴스
new 미사용 → this = 전역 객체
```

---

## 3. 최종 요약

### 콜백 함수

- `this`는 호출 주체가 결정
- 지정 없으면 전역 객체

---

### 생성자 함수

- `new` 사용 시 → `this`는 생성된 객체
- `new` 미사용 시 → `this`는 전역 객체

---

## 4. 핵심 한 줄 정리

- 콜백 함수
→ “this는 내가 정하는 게 아니라, 실행하는 쪽이 정한다”
- 생성자 함수
→ “new를 쓰는 순간 this는 새로 생성된 객체가 된다”
