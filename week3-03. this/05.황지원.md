# 3주차

## 1. 상황에 따라 달라지는 this

- **전역 공간의 this**
    - 전역 객체를 가리킴. 브라우저 → window, Node.js → global
    - 전역변수를 선언하면 자동으로 전역객체의 프로퍼티로 할당하기 때문
    - window에 직접 할당해도 전역 변수로 선언하는 것과 똑같음
    - but, 전역변수로 선언하면 삭제는 안됨
- **메서드로서 호출할 때 그 내부에서의 this**
    - 메서드 : 함수가 객체의 프로퍼티에 할당되어 그 객체의 메서드로서 호출될 경우에만 메서드, 아닐때는 함수로 동작. 구별 방법 → 함수 앞에 . 이 있는지 여부
    - 메서드일 때 this는 호출한 주체에 대한 정보가 담김.
- **함수로서 호출할 때 그 내부에서의 this**
    - 함수를 함수로 호출할 때는 this가 지정되지 않음. (호출 주체가 명시되지 않았기 때문에) → this가 지정되지 않았기 때문에 전역 객체를 바라보게 됨.
    - 함수 / 메서드에서의 this 바인딩은 실행 환경 등은 중요하지 않고 오직 해당 함수 앞에 점이나 대괄호 표기가 있는지로만 결정됨.
    - 메서드 내부에서 호출 주체가 없이 함수를 호출할 때 this가 전역객체가 아닌 주변 환경(상위 스코프)를 가리키도록 우회하기 위해 → 변수를 활용 (보통 self라는 변수명으로 this를 저장해서 사용)
    - 화살표 함수 도입 : 화살표 함수는 실행 컨텍스트를 생성할 때 this 바인딩 자체를 안해서 상위 스코프의 this를 그대로 받아오게 됨.
- **콜백 함수 호출 시 그 내부에서의 this**
    - 함수 A의 제어권을 B에게 넘겨주면 A(콜백함수)의 this는 B의 내부 로직에 따라 결정됨.
    - A도 함수라 기본적으로 전역객체를 참조하지만 B에서 별도로 지정하면 그 대상을 따름.
- **생성자 함수 내부에서의 this**
    - 생성자 함수 : 공통된 성질을 지니는 객체들을 생성하는 데 사용하는 함수 (=클래스)
    - JS에서는 함수를 new 명령어와 호출하면 생성자로서 동작. → 이렇게 호출된 경우 this는 인스턴스 자신

## 2. 명시적으로 this를 바인딩하는 방법

위의 상황별 this 바인딩의 규칙을 깨고 별도로 바인딩하는 방법

- **call 메서드**
    - 메서드의 호출 주체인 함수를 즉시 실행하도록 하는 명령어. → call 메서드의 첫번째 인자를 this로 바인딩
- **apply 메서드**
    - call 메서드와 거의동일. (call과 다른건 두번째 인자를 배열로 받는 것)
- **call / apply 메서드 활용**
    - 객체에는 배열 메서드를 직접 적용 불가 → 유사배열객체에 call/apply 메서드를 이용해 배열 메서드를 차용 (call을 이용해 배열이 아닌 객체에 배열 메서드 slice 사용)
    - 문자열도 배열은 아니지만 call을 이용해 일부 배열 메서드 사용 가능 (concat 등)
    - 생성자 내부에 다른 생성자와 공통된 내용이 있을 경우 call / apply로 반복 제거
- **bind 메서드**
    - call과 비슷하지만 넘겨받은 this 및 인수들로 새로운 함수를 반환하는 메서드 → 다시 새로운 함수를 호출할 때 새로운 인수는 기존 bind에서 전달된 인수 뒤에 이어서 등록됨.
    - bind를 통해 함수의 this 부분과 나머지 인자들을 모두 적용할 수 있음. (부분 적용 함수 구현)
    - bind 메서드를 적용해서 만든 함수는 name 프로퍼티에 ‘bound’라는 접두어가 붙음. (func → bound func)
    - bind를 통해 메서드 내부함수에서 메서드의 this가 전역객체가 아닌 상위스코프를 바라보게 할 수 있음.
    - 콜백 함수를 인자로 받는 함수, 메서드에서 콜백의 this에 관여하는 경우에는 bind를 통해 바꿀 수 있음.
- **화살표 함수**
    - 화살표함수는 this 가 아예 없음. 오히려 메서드 내부함수에서 this 지정이 쉬워짐.
