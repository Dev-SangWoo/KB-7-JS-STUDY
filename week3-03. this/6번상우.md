## 3-1. 상황에 따라 달라지는 this

자바스크립트에서 함수와 객체(메서드)의 구분은 느슨한 편이며, `this`는 실질적으로 이 둘을 구분하는 거의 유일한 기능입니다. **`this`는 기본적으로 실행 컨텍스트가 생성될 때, 즉 '함수를 호출할 때' 결정됩니다.**

### 3-1-1. 전역 공간에서의 this

전역 공간에서 `this`는 전역 객체를 가리킵니다.

- **브라우저 환경:** `window`
- **Node.js 환경:** `global`

전역 변수를 선언하면 자바스크립트 엔진은 이를 전역 객체의 프로퍼티로 할당합니다. 즉, 자바스크립트의 모든 변수는 실은 특정 객체(Lexical Environment, 실행 컨텍스트)의 프로퍼티로서 동작합니다.

> **참고 (`delete` 연산자 관련):** 처음부터 전역 객체의 프로퍼티로 할당한 경우 `delete`로 삭제가 가능하지만, 전역 변수로 선언한 경우에는 `delete` 명령어가 먹히지 않습니다. 이는 JS 엔진이 전역 변수로 선언 시 해당 프로퍼티의 `configurable` 속성을 `false`로 설정하여 의도치 않은 삭제를 방지하기 때문입니다.

```JavaScript
var a = 1;
console.log(window.a); // 1
console.log(this.a);   // 1

window.b = 2;
delete window.b; // true (삭제됨)

var c = 3;
delete window.c; // false (삭제 안 됨)
```

### 3-1-2. 메서드로서 호출할 때 그 메서드 내부에서의 this

함수와 메서드의 가장 큰 차이는 **독립성**입니다.

- **함수:** 그 자체로 독립적인 기능 수행
- **메서드:** 자신을 호출한 대상 객체에 관한 동작 수행

객체의 프로퍼티에 할당된 함수라고 해서 무조건 메서드가 되는 것이 아닙니다. **객체의 메서드로서 호출할 경우(`.`)에만 메서드로 동작하고, 그렇지 않으면 함수로 동작합니다.** 함수 호출과 메서드 호출은 점(`.`) 표기법이나 대괄호(`[]`) 표기법의 유무로 구분할 수 있습니다.

메서드 내부에서의 `this`에는 **호출한 주체에 대한 정보**가 담깁니다. 즉, `.` 앞에 있는 객체가 `this`가 됩니다.

```JavaScript
var obj = {
    method: function (x) { console.log(this, x); }
};

obj.method(1); // this는 obj를 가리킴
var func = obj.method;
func(2);       // 함수로서 호출되었으므로 this는 전역 객체(window/global)를 가리킴
```

### 3-1-3. 함수로서 호출할 때 그 함수 내부에서의 this

어떤 함수를 단순히 함수로서 호출할 경우에는 `this`가 지정되지 않으므로, **자동으로 전역 객체(`window`, `global`)를 가리킵니다.**

**메서드의 내부 함수에서의 this:** 메서드 내부에서 정의하고 실행한 함수라도, 호출하는 구문 앞에 점(`.`)이 없다면 함수로서 호출된 것이므로 `this`는 전역 객체를 바인딩합니다. (호출 당시의 주변 환경은 중요하지 않습니다.)

**메서드 내부 함수에서의 this 우회 방법:**

1. **변수 활용:** 상위 스코프의 `this`를 `self` 등의 변수에 저장하여 내부 함수에서 사용합니다.
2. **화살표 함수:** ES6의 화살표 함수는 `this` 바인딩 과정 자체가 없기 때문에 상위 스코프의 `this`를 그대로 상속받습니다.

```JavaScript
var obj = {
    outer: function() {
        console.log(this); // obj
        
        // 1. 일반 함수 호출 (this = 전역 객체)
        var innerFunc = function() { console.log(this); };
        innerFunc(); // window
        
        // 2. 변수를 활용한 우회 (self 사용)
        var self = this;
        var innerFunc2 = function() { console.log(self); };
        innerFunc2(); // obj
        
        // 3. 화살표 함수 사용 (상위 스코프 this 상속)
        var innerFunc3 = () => { console.log(this); };
        innerFunc3(); // obj
    }
};
obj.outer();
```

### 3-1-4. 콜백 함수 호출 시 그 함수 내부에서의 this

콜백 함수란 함수 A의 제어권을 다른 함수(또는 메서드) B에게 넘겨주는 경우의 함수 A를 말합니다. 콜백 함수도 기본적으로는 함수이기 때문에 `this`가 전역 객체를 참조합니다. 하지만 **제어권을 받은 함수(B)에서 콜백 함수에 별도로 `this`가 될 대상을 지정한 경우**에는 그 대상을 참조합니다.

```JavaScript
// 1. 기본적으로 전역 객체를 참조
setTimeout(function() { console.log(this); }, 300); // window

// 2. addEventListener는 내부적으로 자신의 this를 콜백의 this로 바인딩하도록 정의되어 있음
document.body.innerHTML += '<button id="a">클릭</button>';
document.body.querySelector('#a').addEventListener('click', function(e) {
    console.log(this); // 지정된 엘리먼트 (<button id="a">클릭</button>)
});
```

### 3-1-5. 생성자 함수 내부에서의 this

생성자 함수란 어떤 공통된 성질을 지니는 객체(인스턴스)들을 생성하는 데 사용하는 일종의 '틀'입니다. `new` 명령어와 함께 함수를 호출하면 해당 함수가 생성자로서 동작하게 되며, 이때 **내부에서의 `this`는 곧 새로 만들어질 구체적인 인스턴스 자신**이 됩니다.

```JavaScript
var Cat = function(name, age) {
    this.bark = '야옹';
    this.name = name;
    this.age = age;
};

var choco = new Cat('초코', 7);
var nabi = new Cat('나비', 5);

console.log(choco); // Cat { bark: '야옹', name: '초코', age: 7 }
console.log(nabi);  // Cat { bark: '야옹', name: '나비', age: 5 }
```

## 3-2. 명시적으로 this를 바인딩하는 방법

앞선 규칙들(자동 바인딩)을 깨고 `this`를 명시적으로 지정하는 방법들입니다.

### 3-2-1. call 메서드

`call` 메서드는 메서드의 호출 주체인 함수를 즉시 실행하도록 하는 명령어입니다. **첫 번째 인자를 `this`로 바인딩**하고, 이후의 인자들을 호출할 함수의 매개변수로 사용합니다.

```JavaScript
var func = function(a, b, c) {
    console.log(this, a, b, c);
};

func(1, 2, 3);                 // window 1 2 3
func.call({x: 1}, 4, 5, 6);    // {x: 1} 4 5 6
```

### 3-2-2. apply 메서드

`apply` 메서드는 `call` 메서드와 기능적으로 완전히 동일합니다. 단지 두 번째 인자를 **배열**로 받아, 그 배열의 요소들을 호출할 함수의 매개변수로 지정한다는 점만 다릅니다.


```JavaScript
func.apply({x: 1}, [4, 5, 6]); // {x: 1} 4 5 6
```

### 3-2-3. call / apply 메서드의 활용

가장 대표적인 활용처는 **유사 배열 객체(Array-like Object)**에 배열 메서드를 적용할 때입니다. 키가 0 또는 양의 정수인 프로퍼티가 존재하고, `length` 프로퍼티 값이 0 또는 양의 정수인 객체(`arguments`, `NodeList` 등)에서 `call`이나 `apply`를 이용해 배열 메서드를 차용할 수 있습니다.

```JavaScript
var obj = { 0: 'a', 1: 'b', 2: 'c', length: 3 };

// 유사 배열 객체에 배열의 push 메서드 적용
Array.prototype.push.call(obj, 'd');
console.log(obj); // { 0: 'a', 1: 'b', 2: 'c', 3: 'd', length: 4 }

// 유사 배열 객체를 진짜 배열로 변환
var arr = Array.prototype.slice.call(obj);
console.log(arr); // ['a', 'b', 'c', 'd']
```

### 3-2-4. bind 메서드

`call`과 비슷하지만 **즉시 호출하지 않고, 넘겨받은 `this` 및 인수들을 바탕으로 새로운 함수를 반환**하기만 하는 메서드입니다. 
목적: 함수에 `this`를 미리 적용하기, 부분 적용 함수(Partial Application) 구현하기.

```JavaScript
var func = function(a, b, c, d) {
    console.log(this, a, b, c, d);
};

// this를 {x: 1}로 지정하고 앞에서부터 두 개의 인자(4, 5)를 미리 적용
var bindFunc = func.bind({x: 1}, 4, 5); 

bindFunc(6, 7); // {x: 1} 4 5 6 7
```

### 3-2-5. 화살표 함수의 예외사항

ES6에 도입된 화살표 함수는 실행 컨텍스트 생성 시 `this`를 바인딩하는 과정 자체가 제외되었습니다. 따라서 이 함수 내부에는 `this`가 아예 없으며, 접근하고자 하면 스코프 체인상 가장 가까운 상위 컨텍스트의 `this`를 그대로 참조합니다.

```JavaScript
var obj = {
    outer: function() {
        var innerFunc = () => {
            // this 바인딩 과정이 없으므로 상위 스코프인 outer의 this(obj)를 참조
            console.log(this); 
        };
        innerFunc();
    }
};
obj.outer(); // obj
```

### 3-2-6. 별도의 인자로 this를 받는 경우 (콜백 함수 내에서의 this)

요소를 순회하면서 콜백 함수를 반복 호출하는 내용의 일부 배열 메서드(`forEach`, `map`, `filter` 등)는 콜백 함수 내부에서 사용할 `this`를 별도의 인자로 지정할 수 있습니다.

```JavaScript
var report = {
    sum: 0,
    count: 0,
    add: function() {
        var args = Array.prototype.slice.call(arguments);
        // forEach의 두 번째 인자로 this(report 객체)를 넘겨줌
        args.forEach(function(entry) {
            this.sum += entry;
            this.count++;
        }, this); 
    }
};

report.add(60, 85, 95);
console.log(report.sum, report.count); // 240 3
```

## 3-3. 정리

### this의 기본 규칙 (호출 방식에 따른 결정)

1. **전역 공간에서의 this:** 전역 객체(`window` 또는 `global`)를 참조합니다.
    
2. **메서드로서 호출한 경우:** 메서드 호출 주체(`.` 앞의 객체)를 참조합니다.
    
3. **함수로서 호출한 경우:** 전역 객체를 참조합니다. (메서드의 내부 함수에서도 동일)
    
4. **콜백 함수 내부에서의 this:** 해당 콜백 함수의 제어권을 넘겨받은 함수가 정의한 바에 따르며, 특별히 정의하지 않은 경우에는 전역 객체를 참조합니다.
    
5. **생성자 함수에서의 this:** 생성될 인스턴스 자신을 참조합니다.
    

### this의 명시적 바인딩

1. **`call`, `apply` 메서드:** `this`를 명시적으로 지정하면서 함수 또는 메서드를 즉시 호출합니다.
    
2. **`bind` 메서드:** `this` 및 함수에 넘길 인수를 일부 지정해서 새로운 함수를 반환합니다.
    
3. **배열 순회 메서드 등:** 요소를 순회하면서 콜백 함수를 반복 호출하는 일부 메서드(`forEach` 등)는 별도의 인자로 `this`를 받기도 합니다.
