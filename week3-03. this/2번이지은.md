3-1-3. 함수로서 호출할 때 그 내부에서의 this
1. 함수 호출과 this
어떤 함수를 '함수로서' 호출할 경우, this는 전역 객체를 바라봅니다. 이는 설계상의 오류라고 지적받기도 하는 자바스크립트의 독특한 특징입니다.

브라우저 환경: window

Node.js 환경: global

2. 메서드 내부 함수에서의 this (설계상의 오류)
메서드 내부에서 정의하고 호출한 함수라도, 그것을 '함수로서' 호출하면 this는 전역 객체를 바인딩합니다.

❌ 문제 상황
JavaScript
var obj1 = {
  outer: function () {
    console.log(this); // (1) obj1 (메서드 호출)
    
    var innerFunc = function () {
      console.log(this); // (2) Window (함수 호출)
    };
    innerFunc(); 
  }
};
obj1.outer();
(1)은 obj1.outer()로 호출되었으므로 this가 점(.) 앞의 객체인 obj1이 됩니다.

(2)는 innerFunc()로 호출되었습니다. 비록 outer 메서드 내부에서 실행되었지만, 독립적인 함수로서 호출되었기 때문에 this는 전역 객체를 가리키게 됩니다.

더글라스 크락포드(Douglas Crockford)의 비판: > 호출 주체가 없을 때는 주변의 this를 상속받는 것이 논리적으로 당연해 보이지만, 자바스크립트는 이를 전역 객체에 바인딩함으로써 '설계상의 결함'을 가지게 되었습니다.

3. this 바인딩을 우회하는 방법
메서드 내부 함수에서 주변 컨텍스트의 this를 그대로 유지하고 싶을 때 사용하는 전통적인 방법입니다.

✅ 변수 활용 (self, that, _this)
상위 스코프의 this를 변수에 저장하여 내부 함수에서 클로저를 통해 접근하는 방식입니다.

JavaScript
var obj = {
  outer: function () {
    console.log(this); // obj
    var self = this;   // 상위 스코프의 this를 변수에 저장
    
    var innerFunc = function () {
      console.log(self); // obj (변수를 통해 상위 this에 접근)
    };
    innerFunc();
  }
};
obj.outer();
변수명은 관례적으로 self, that, _this 등을 사용합니다.

4. ES6의 해결책: 화살표 함수 (Arrow Function)
ES6에서 도입된 화살표 함수는 실행 컨텍스트 생성 시 this 바인딩 과정 자체가 빠져 있습니다. 즉, 상위 스코프의 this를 그대로 활용합니다.

JavaScript
var obj = {
  outer: function () {
    console.log(this); // obj
    
    var innerFunc = () => {
      console.log(this); // obj (상위 스코프의 this를 그대로 사용)
    };
    innerFunc();
  }
};
obj.outer();
