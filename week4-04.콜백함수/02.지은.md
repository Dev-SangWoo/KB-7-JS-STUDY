## 4-2-2. 인자 (Arguments)

콜백 함수의 제어권을 넘겨받은 코드는 콜백 함수를 호출할 때 인자에 **어떤 값들을 어떤 순서로 넘길 것인지**에 대한 제어권을 가집니다.

### 1. `Array.prototype.map` 분석

`map` 메서드는 배열의 모든 요소를 처음부터 끝까지 하나씩 꺼내어 콜백 함수를 반복 호출하고, 그 실행 결과들을 모아 새로운 배열을 만듭니다.

- **기본 구조**

```jsx
01 var newArr2 = [10, 20, 30].map(function (currentValue, index) {
02 console.10g(currentValue, index) ;
03 return currentValue + 5;
04 ));
05 console. log(newArr2) ;
// [실행결과]
// 10 0
// 20 1
// 30 2
// [15, 25, 35]
```

- **`callback`**: 각 요소에 대해 실행할 함수로, 세 가지 인자를 전달받습니다.
    - **`currentValue`**: 현재 처리되고 있는 요소의 값.
    - **`index`**: 현재 처리되고 있는 요소의 인덱스.
    - **`array`**: `map`을 호출한 원본 배열 자체.
- **`thisArg` (선택 사항)**: 콜백 함수 내부에서 `this`로 사용할 값입니다.

### 2. 인자의 순서와 명명

컴퓨터는 매개변수의 이름이 아니라 '순서'에 의해서만 각 인자를 구분합니다.

```jsx
var newArr = [10, 20, 30].map(function (index, currentValue) {
    console.log(index, currentValue);
    return currentValue + 5;
});

console.log(newArr);

// [실행 결과]
// 10 0  (첫 번째 인자 index 자리에 요소의 값 10이 들어옴)
// 20 1  (첫 번째 인자 index 자리에 요소의 값 20이 들어옴)
// 30 2  (첫 번째 인자 index 자리에 요소의 값 30이 들어옴)
// [5, 6, 7] (두 번째 인자 currentValue 자리에 들어온 인덱스 0, 1, 2에 5를 더한 결과)
```

사용자가 첫 번째 인자 이름을 `index`, 두 번째를 `currentValue`로 지었다고 해서 인덱스가 먼저 들어오지 않습니다.

- **첫 번째 자리:** 무조건 **현재 요소의 값(Value)**
- **두 번째 자리:** 무조건 **현재 요소의 인덱스(Index)**

## 4-2-3. this

콜백 함수도 함수이기 때문에 기본적으로는 `this`가 전역객체를 참조하지만, 제어권을 넘겨받을 코드에서 별도로 `this`가 될 대상을 지정한 경우에는 그 대상을 참조하게 됩니다.

### 1. `map` 메서드의 내부 구현

`map` 메서드가 내부적으로 `call` 또는 `apply`를 사용하여 `this`를 바인딩하는 원리는 다음과 같습니다.

```jsx
Array.prototype.map = function (callback, thisArg) {
    var mappedArr = [];
    for (var i = 0; i < this.length; i++) {
        // call을 사용해 thisArg가 있으면 그것을, 없으면 전역객체를 this로 지정
        var mappedValue = callback.call(thisArg || window, this[i], i, this);
        mappedArr[i] = mappedValue;
    }
    return mappedArr;
};
```

### 2. 제어권에 따른 `this`의 변화

각 메서드는 내부 설계에 따라 콜백 함수에 넘겨줄 `this`를 결정합니다.

```jsx
// 1. setTimeout (단순 호출)
var setTimeout = function (callback, delay) {
    // ... 시간 지연 로직 ...
    callback(); // 일반 함수 호출이므로 this는 전역객체
};

// 2. forEach (thisArg 활용)
Array.prototype.forEach = function (callback, thisArg) {
    for (var i = 0; i < this.length; i++) {
        callback.call(thisArg || window, this[i], i, this);
    }
};

// 3. addEventListener (이벤트 타겟 바인딩)
var addEventListener = function (type, callback) {
    var self = this; // 엘리먼트 자체
    // ... 이벤트 발생 시 ...
    callback.call(self, event); // 인자로 받은 콜백을 해당 엘리먼트에 바인딩하여 호출
};
```

| **메서드** | **콜백 내부의 this** | **특징** |
| --- | --- | --- |
| **setTimeout** | `Window` | 별도의 `this` 지정 과정이 없음 |
| **forEach** | `Window` (또는 지정 대상) | `thisArg`를 넘기지 않으면 전역객체 참조 |
| **addEventListener** | **이벤트가 발생한 엘리먼트** | 호출 시 `call` 메서드의 첫 번째 인자에 `this`를 넘기도록 정의됨 |
