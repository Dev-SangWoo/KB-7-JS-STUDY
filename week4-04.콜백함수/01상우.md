# 📚 4장: 콜백 함수 (Callback Function)

### 1. 콜백함수란?

> **다른 코드의 인자로 넘겨주는 함수 (매개변수에 들어가는 함수)**

콜백 함수를 넘겨받은 코드는 이 콜백 함수를 필요에 따라 **적절한 시점에 실행**합니다. 어떤 함수 X를 호출하면서 "특정 조건일 때 함수 Y를 실행해서 나에게 알려달라"는 요청을 함께 보내는 것입니다. 이 요청을 받은 함수 X의 입장에서는 해당 조건이 갖춰졌는지 여부를 스스로 판단하고 함수 Y를 직접 호출합니다.

- **의미:** callback은 '되돌아 호출하다', '다시 호출해달라'는 명령입니다.  
    
- **제어권 위임:** 즉, 콜백 함수는 다른 코드(함수 또는 메서드)에게 인자로 넘겨짐으로써 **그 제어권도 함께 위임**한 함수입니다. (예: 시계 함수에게 알람을 울리는 명령에 대한 제어권을 넘겨주어, 언제 함수를 실행할지 결정하게 함)  
    

---

### 2. 제어권

콜백 함수를 넘겨받은 코드는 콜백 함수를 호출할 시점과 인자에 대한 제어권을 가집니다.

```javascript
var count = 0;
var cbFunc = function() {
    console.log(count);
    if (++count > 4) clearInterval(timer);
};

// setInterval이 cbFunc의 실행 제어권을 가짐 (300ms 마다)
var timer = setInterval(cbFunc, 300);
```

**인자 (Arguments)의 규칙** 컴퓨터는 매개변수를 이름으로 인식하는 것이 아니라, 단지 **첫 번째, 두 번째라는 순서**에 의해서만 각각을 구분하고 인식합니다.

- 원하는 시각에 알람이 울리는 결과를 얻으려면 시계가 정한 규칙(알람 스위치 ON 등)을 따라야 합니다.  
    
- 마찬가지로 `map` 메서드를 호출해서 원하는 배열을 얻으려면, `map` 메서드에 정의된 규칙(인자의 순서)에 따라 콜백 함수를 작성해야 합니다.  
    

---

### 3. 콜백 함수는 함수다.

어떤 함수의 인자에 객체의 메서드를 전달하더라도, 이는 결국 메서드가 아닌 **함수일 뿐**입니다. 결국 다른 함수의 인자로 들어가는 콜백 함수는 (원래 메서드였더라도) 모두 함수로서 작동합니다. 이는 별도의 처리가 없다면 **내부의 `this`가 전역 객체로 묶인다**는 뜻입니다.

---

### 4. 콜백 함수 내부의 this에 다른 값 바인딩하기

객체의 메서드를 콜백 함수로 전달할 때 `this`를 유지하기 위한 방법들입니다.

**방법 1: 클로저와 `call()` 활용 (전통적 방식)** 전통적으로는 `this`를 다른 변수(`self`)에 담아 콜백 함수로 활용할 함수에서 `this` 대신 그 변수를 사용하게 하고, 이를 클로저로 만드는 방식이 많이 쓰였습니다.

```javascript
var obj1 = {
    name: "obj1",
    func: function() {
        var self = this;
        return function() {
            console.log(self.name);
        }
    }
};
var callback = obj1.func();
setTimeout(callback, 1000);

var obj2 = {
    name: "obj2",
    func: obj1.func 
};
var callback2 = obj2.func();
setTimeout(callback2, 1500);

// call() 메서드로 this를 명시하여 바인딩
var obj3 = { name: "obj3" };
var callback3 = obj1.func.call(obj3);
setTimeout(callback3, 2000);
```

**방법 2: `bind()` 메서드 활용 (권장 방식)** 최근에는 `.bind()` 메서드를 활용하여 훨씬 깔끔하게 `this`를 바인딩합니다.

```javascript
var obj1 = {
    name: "obj1",
    func: function() {
        console.log(this.name);
    }
};
setTimeout(obj1.func.bind(obj1), 1000);

var obj2 = { name: "obj2" };
setTimeout(obj1.func.bind(obj2), 1500);
```

### 5. 콜백 지옥과 비동기 제어

- **동기적 코드:** 현재 실행 중인 코드가 완료된 후에야 다음 코드를 실행하는 방식. (CPU 계산 등 즉시 처리 가능한 대부분의 코드)  
    
- **비동기적 코드:** 현재 실행 중인 코드의 완료 여부와 무관하게 즉시 다음 코드로 넘어가는 방식. (사용자의 요청 대기, 시간 지연, 외부 API 응답 대기 등 별도의 보류/대기가 필요한 코드)  
    

**콜백 지옥 (Callback Hell)** 콜백 함수를 익명 함수로 전달하는 과정이 반복되어, 코드의 들여쓰기 수준이 감당하기 힘들 정도로 깊어지는 현상입니다.

```javascript
// 콜백 지옥 예시
setTimeout(function (name) {
    var coffeeList = name;
    console.log(coffeeList);
    
    setTimeout(function(name) {
        coffeeList += ', ' + name;
        console.log(coffeeList);
        
        setTimeout(function(name) {
            coffeeList += ', ' + name;
            console.log(coffeeList);
            
            setTimeout(function(name) {
                coffeeList += ', ' + name;
                console.log(coffeeList);
            }, 500, "카페모카");
        }, 500, "아메리카노");
    }, 500, "카페라떼");
}, 500, "에스프레소");
```

**해결 방법 1: 기명 함수로 변환**

```javascript
var coffeeList = '';

var addEspresso = function(name) {
    coffeeList = name;
    console.log(coffeeList);
    setTimeout(addAmericano, 500, "아메리카노");
};

var addAmericano = function(name) {
    coffeeList += ', ' + name;
    console.log(coffeeList);
    setTimeout(addLatte, 500, "카페라떼");
};

var addLatte = function(name) {
    coffeeList += ', ' + name;
    console.log(coffeeList);
};

setTimeout(addEspresso, 500, "에스프레소");
```

**해결 방법 2: Promise**

```javascript
// Promise 1: 기본적인 체이닝
new Promise(function (resolve) {
    setTimeout(function() {
        var name = '에스프레소';
        console.log(name);
        resolve(name); // resolve라는 콜백함수에 name이라는 값을 넣어주기
    }, 500);
}).then(function (prevName) {  // then: 위에꺼 하고 나서 아래꺼 할꺼얌 (비동기를 동기적으로)
    return new Promise(function (resolve) {
        setTimeout(function() {
            var name = prevName + ', 아메리카노';
            console.log(name);
            resolve(name); 
        }, 500);
    });
});

// Promise 2: 함수화하여 재사용
var addCoffee = function(name) {
    return function(prevName) {
        return new Promise(function (resolve) {
            setTimeout(function() {
                var newName = prevName ? (prevName + ', ' + name) : name;
                console.log(newName);
                resolve(newName);
            }, 500);
        });
    }
};

addCoffee('에스프레소')()
    .then(addCoffee('아메리카노'))
    .then(addCoffee('카페모카'));
```

**해결 방법 3: Async/Await** 전체를 순서대로 실행해 주는 `async` 부분이 있고, 그 안에서 각 흐름을 `await`로 동기화해 주는 안에서 굴러갑니다.

```javascript
// 커피 이름을 Promise로 반환하는 함수
var addCoffee = function (name) {
    return new Promise(function (resolve) {
        setTimeout(function () {
            resolve(name);
        }, 500);
    });
};

// 커피를 순서대로 만드는 비동기 함수
var coffeeMaker = async function () {
    var coffeeList = '';

    // 커피 하나를 추가하는 내부 함수
    var _addCoffee = async function (name) {
        coffeeList += (coffeeList ? ', ' : '') + await addCoffee(name);
    };

    // coffeeMaker의 async 안에서 await로 동기적 실행 보장
    await _addCoffee('에스프레소');
    console.log(coffeeList);

    await _addCoffee('아메리카노');
    console.log(coffeeList);

    await _addCoffee('카페모카');
    console.log(coffeeList);

    await _addCoffee('카페라떼');
    console.log(coffeeList);
};

// 실행
coffeeMaker();
```

### 6. 정리

1. **제어권 위임:** 콜백 함수는 다른 코드에 인자로 넘겨줌으로써 그 제어권도 함께 위임한 함수입니다.
    
2. **제어권의 종류:** 제어권을 넘겨받은 코드는 다음과 같은 제어권을 가집니다.
    
    - 콜백 함수를 **호출하는 시점**을 스스로 판단해서 실행합니다.  
        
    - 콜백 함수를 호출할 때 **인자로 넘겨줄 값들 및 그 순서**가 정해져 있습니다. 이 순서를 따르지 않으면 엉뚱한 결과를 얻게 됩니다.  
        
3. **`this` 바인딩:** 콜백 함수의 `this`가 무엇을 바라보도록 할지가 정해져 있는 경우도 있습니다. 정하지 않은 경우에는 전역 객체를 바라봅니다. 사용자가 임의로 `this`를 바꾸고 싶은 경우 `bind` 메서드를 활용하면 됩니다.
    
4. **함수로서의 실행:** 어떤 함수에 인자로 객체의 메서드를 전달하더라도, 이는 결국 메서드가 아닌 '함수'로서 실행됩니다.
    
5. **비동기 제어:** 비동기 제어를 위해 콜백 함수를 사용하다 보면 콜백 지옥에 빠지기 쉽습니다. 최근의 ECMAScript에는 `Promise`, `Generator`, `async/await` 등 콜백 지옥에서 벗어날 수 있는 효율적인 방법들이 속속 등장하고 있습니다.
