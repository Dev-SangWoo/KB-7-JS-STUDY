# 콜백 함수는 함수다

- 콜백 함수는 함수임 → 객체의 메서드를 전달하더라도 메서드가 아닌 **함수로서 호출**됨

```js
var obj = {
  vals: [1, 2, 3],
  logValues: function (v, i) {
    console.log(this, v, i);
  },
};

obj.logValues(1, 2); // this = obj (메서드로 호출)
[4, 5, 6].forEach(obj.logValues); // this = window (함수로 호출)
```

- `obj.logValues(1, 2)` : 점 앞에 obj → 메서드로 호출 → this = obj
- `forEach(obj.logValues)` : obj.logValues를 값으로 꺼내서 넘긴 것 → **점(.)이 없어지는 순간 메서드가 아닌 함수 호출로 바뀜** → this = window

> **점(.)이 없어지는 순간 메서드 → 함수로 바뀌고 this = window**
> 어떤 함수의 인자에 객체의 메서드를 전달하더라도 이는 결국 메서드가 아닌 함수일 뿐

## 콜백 함수에서 this를 객체로 바라보게 하는 방법

**ES5 방식 - self에 this 저장**

```js
var obj1 = {
  name: 'obj1',
  func: function () {
    var self = this; // this = obj1 저장
    return function () {
      console.log(self.name); // 클로저로 obj1 참조
    };
  },
};
var callback = obj1.func();
setTimeout(callback, 1000); // 'obj1' 출력
```

- 콜백은 함수 호출이라 this = window가 되지만
- `self`에 미리 obj1을 저장해두고 클로저로 참조하는 방식
- 번거롭고 this를 직접 쓰지 않아서 가독성이 떨어짐

**ES6 방식 - 화살표 함수 (현재 주로 사용)**

```js
var obj1 = {
  name: 'obj1',
  func: function () {
    setTimeout(() => {
      console.log(this.name); // 상위 스코프(func)의 this = obj1
    }, 1000);
  },
};
obj1.func(); // 'obj1' 출력
```

- 화살표 함수는 자신만의 this가 없고 상위 스코프의 this를 그대로 참조
- self 같은 우회 없이 바로 this 사용 가능

## this를 직접 쓰지 않으면 재활용 불가

```js
// this 안 쓰고 obj1 직접 참조
var obj1 = {
  name: 'obj1',
  func: function () {
    console.log(obj1.name); // obj1에 하드코딩
  },
};
setTimeout(obj1.func, 1000); // 'obj1' 출력

// obj2에서 재활용 시도
var obj2 = { name: 'obj2', func: obj1.func };
setTimeout(obj2.func, 1000); // 'obj2' 아니고 'obj1' 출력 → 재활용 불가
```

- obj1을 직접 참조하면 간결하지만 **obj1에 고정**되어 버림
- obj2를 선언하고 name을 출력하려 해도 obj2.name이 아닌 **obj1.name이 출력**됨

> **this를 사용하면 재활용 가능, 객체를 직접 참조하면 재활용 불가**

## bind로 this 고정 (ES5)

```js
var obj1 = {
  name: 'obj1',
  func: function () {
    console.log(this.name);
  },
};

setTimeout(obj1.func.bind(obj1), 1000); // 1초 뒤 'obj1'

var obj2 = { name: 'obj2' };
setTimeout(obj1.func.bind(obj2), 1500); // 1.5초 뒤 'obj2'
```

- `bind(obj1)` → this를 obj1으로 고정한 새 함수 반환
- self 방식보다 훨씬 간결하게 this 고정 가능
- self 방식, bind 방식 모두 ES5 → 요즘은 화살표 함수로 대체
