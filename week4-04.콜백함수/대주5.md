# 콜백 지옥과 비동기 제어 해결 (Promise, Generator, async/await)

## 1. 개요

콜백 함수는 비동기 처리를 위해 자주 사용되지만,

여러 개가 중첩되면 코드가 복잡해지는 문제가 발생한다.

이러한 현상을 **콜백 지옥(callback hell)**이라고 하며,

가독성과 유지보수 측면에서 큰 단점이 있다.

이를 해결하기 위해 자바스크립트는 다음과 같은 방식으로 발전해왔다.

- Promise (ES6)
- Generator (ES6)
- async/await (ES2017)

---

## 2. Generator를 이용한 해결

### 2.1 개념

Generator는 함수 실행을 중간에 멈췄다가 다시 이어서 실행할 수 있는 기능이다.

- `yield` : 실행을 멈추는 지점
- `next()` : 실행을 다시 이어가는 함수

---

### 2.2 동작 방식

```
varaddCoffee=function (prevName,name) {
setTimeout(function () {
coffeeMaker.next(prevName?prevName+', '+name:name);
  },500);
};

varcoffeeGenerator=function* () {
varespresso=yieldaddCoffee('','에스프레소');
console.log(espresso);

varamericano=yieldaddCoffee(espresso,'아메리카노');
console.log(americano);
};

varcoffeeMaker=coffeeGenerator();
coffeeMaker.next();
```

---

### 2.3 특징

- 비동기 흐름을 순차적으로 제어 가능
- 콜백 함수 중첩 제거 가능

---

### 2.4 한계

- `next()`를 직접 호출해야 함
- 구조가 복잡하고 직관적이지 않음
- 실무에서 거의 사용되지 않음

---

## 3. async/await를 이용한 해결

### 3.1 개념

async/await는 Promise를 기반으로 한 문법으로,

비동기 코드를 동기 코드처럼 작성할 수 있게 해준다.

---

### 3.2 동작 방식

```
varaddCoffee=function (name) {
returnnewPromise(function (resolve) {
setTimeout(function () {
resolve(name);
    },500);
  });
};

varcoffeeMaker=asyncfunction () {
varcoffeeList='';

var_addCoffee=asyncfunction (name) {
coffeeList+= (coffeeList?', ':'')+awaitaddCoffee(name);
  };

await_addCoffee('에스프레소');
console.log(coffeeList);

await_addCoffee('아메리카노');
console.log(coffeeList);
};
```

---

### 3.3 특징

- `await`를 통해 비동기 작업 완료까지 대기
- 코드 흐름이 위에서 아래로 자연스럽게 실행됨
- 가독성이 매우 높음

---

### 3.4 장점

- 콜백 지옥 문제 해결
- Promise보다 직관적인 코드 작성 가능
- 유지보수 용이

---

## 4. 방식 비교

| 방식 | 특징 | 단점 |
| --- | --- | --- |
| 콜백 함수 | 기본 비동기 처리 방식 | 콜백 지옥 발생 |
| Promise | then 체인으로 흐름 제어 | 체인이 길어질 수 있음 |
| Generator | 실행 흐름 직접 제어 | 구조 복잡 |
| async/await | 동기 코드처럼 작성 가능 | Promise 이해 필요 |

---

## 5. 결론

콜백 함수는 비동기 처리의 기본 방식이지만,

중첩 구조로 인해 코드가 복잡해지는 문제가 있다.

이를 해결하기 위해 Promise, Generator, async/await가 등장했으며,

현재는 **async/await 방식이 가장 널리 사용된다.**

---

## 핵심 요약

- 콜백 지옥은 가독성과 유지보수 문제를 발생시킨다
- Promise는 이를 부분적으로 해결한다
- Generator는 흐름 제어가 가능하지만 복잡하다
- async/await는 가장 직관적이고 효율적인 해결 방식이다
