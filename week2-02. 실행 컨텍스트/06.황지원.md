# 2주차

## 1. 실행 컨텍스트란

> **코드를 실행하기 위해 필요한 환경 정보들을 모아놓은 객체**
> 

자바스크립트 엔진은 코드를 실행할 때 단순히 위에서부터 읽는 것이 아니라,

**실행 컨텍스트를 생성하고 그 안에서 변수, 함수, 스코프, this 등을 관리한다.**

---

## 2. 실행 컨텍스트의 구성 요소

실행 컨텍스트는 크게 3가지로 구성된다.

### 1) Lexical Environment

> 현재 실행 중인 코드의 **스코프와 변수 정보를 관리**
> 

구성:

- **Environment Record**: 현재 스코프의 변수/함수 저장
- **Outer Environment Reference**: 외부 스코프 참조 (스코프 체인)

---

### 2) Variable Environment

> 실행 컨텍스트 생성 시점의 **초기 상태 스냅샷**
> 
- 구조는 Lexical Environment와 동일
- 이후 실제 실행에서는 **Lexical Environment가 사용됨**

---

### 3) This Binding

> 현재 실행 컨텍스트에서의 `this` 값
> 
- 전역: window (브라우저)
- 함수 호출: 기본적으로 window
- 메서드 호출: 호출한 객체

---

## 3. Lexical Environment 구조

```jsx
let a = 1;

function foo() {
  let b = 2;
  console.log(a);
}
```
