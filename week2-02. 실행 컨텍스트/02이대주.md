# LexicalEnvironment와 변수 호이스팅 정리

## 1. LexicalEnvironment

LexicalEnvironment는 현재 실행 컨텍스트의 환경 정보를 담고 있는 객체이다.

코드가 실행될 때 필요한 변수, 함수, 그리고 외부 환경에 대한 정보를 관리한다.

### 구성 요소

### 1) environmentRecord

현재 실행 컨텍스트 내부에 존재하는 식별자 정보를 저장한다.

- 매개변수
- 변수 (var)
- 함수 선언

즉, 현재 스코프 내부의 모든 변수와 함수 정보가 저장된다.

---

### 2) outerEnvironmentReference

외부 환경을 참조하는 역할을 한다.

현재 스코프에서 변수를 찾을 수 없을 경우,

이 참조를 통해 상위 스코프로 이동하여 변수를 탐색한다.

---

### 정리

LexicalEnvironment는 다음과 같이 구성된다.

- 현재 스코프의 변수 목록
- 외부 스코프에 대한 참조

---

## 2. environmentRecord와 호이스팅

자바스크립트는 코드를 실행하기 전에

현재 실행 컨텍스트에 포함된 변수와 함수들을 먼저 수집한다.

이 과정은 실제 동작을 설명하기 위해

"호이스팅"이라는 개념으로 표현된다.

---

## 3. 호이스팅

호이스팅은 변수와 함수 선언이

코드의 최상단으로 끌어올려진 것처럼 동작하는 현상이다.

실제로 코드가 이동하는 것은 아니며,

자바스크립트 엔진이 실행 전에 선언을 먼저 처리하기 때문에 발생한다.

---

## 4. 호이스팅 동작 방식

### 1) 변수 (var)

```
console.log(a);
vara=10;
```

위 코드는 다음과 같이 동작한다.

```
vara;
console.log(a);// undefined
a=10;
```

정리하면

- 변수 선언은 먼저 처리된다.
- 값 할당은 원래 위치에서 실행된다.

---

### 2) 매개변수

```
functiona(x) {
console.log(x);
}
a(1);
```

매개변수는 함수 실행 시 이미 선언과 동시에 값이 할당된 상태로 시작한다.

즉, 매개변수는 내부적으로 미리 준비된 변수로 동작한다.

---

### 3) 중복 선언

```
varx;
varx;
```

이미 선언된 변수는 다시 선언해도 무시된다.

---

### 4) 예제

```
functiona(x) {
console.log(x);
varx;
console.log(x);
varx=2;
console.log(x);
}
a(1);
```

실행 결과

```
1
1
2
```

설명

- 선언은 이미 호이스팅으로 처리된 상태
- 중간의 `var x`는 영향 없음
- 값만 변경되면서 결과가 달라짐

---

## 5. 함수 호이스팅

### 1) 함수 선언문

```
console.log(sum(1,2));

functionsum(a,b) {
returna+b;
}
```

정상적으로 실행된다.

이유는 함수 선언문은 전체가 호이스팅되기 때문이다.

---

### 2) 함수 표현식

```
console.log(multiply(2,3));

varmultiply=function(a,b) {
returna*b;
}
```

에러가 발생한다.

이유는 변수 선언만 호이스팅되고

함수 할당은 이후에 이루어지기 때문이다.

---

### 3) 차이 정리

| 구분 | 호이스팅 동작 |
| --- | --- |
| 함수 선언문 | 전체가 호이스팅 |
| 함수 표현식 | 변수 선언만 호이스팅 |

---

## 6. 결론

- 자바스크립트는 코드 실행 전에 변수와 함수를 먼저 수집한다.
- 변수는 선언만 먼저 처리되고, 값은 이후에 할당된다.
- 함수 선언문은 전체가 호이스팅된다.
- 함수 표현식은 선언만 호이스팅된다.

실무에서는 예측 가능성과 안정성을 위해

함수 표현식을 사용하는 것이 더 안전하다.

---

## 핵심 요약

- LexicalEnvironment는 실행 환경 정보 묶음이다.
- environmentRecord는 현재 스코프의 변수 정보를 저장한다.
- outerEnvironmentReference는 외부 스코프를 참조한다.
- 호이스팅은 선언이 먼저 처리되는 동작을 설명하는 개념이다.
- 함수 선언문과 함수 표현식은 호이스팅 방식이 다르다.
