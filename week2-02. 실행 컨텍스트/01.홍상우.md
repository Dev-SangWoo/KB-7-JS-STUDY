## 1. 실행 컨텍스트(Execution Context)란?

실행 컨텍스트는 **"실행할 코드에 제공할 환경 정보들을 모아놓은 객체"**입니다. 자바스크립트 엔진이 코드를 실행하기 위해 필요한 일종의 '배경 정보'라고 생각하면 쉽습니다.

- **역할:** 선언된 변수, 함수, 스코프, `this` 등을 관리합니다.
    
- **생성 시점:** 우리가 실행 컨텍스트를 구성하는 가장 일반적인 방법은 **함수를 실행하는 것**입니다. (그 외에는 전역 공간, `eval()` 함수 등이 있습니다.)
    

---

## 2. 콜 스택(Call Stack)과 실행 순서

자바스크립트는 동일한 환경에 있는 코드들을 실행할 때 필요한 정보들을 모아 컨텍스트를 만들고, 이를 **콜 스택(Call Stack)**에 쌓아 올립니다.

### 📚 스택(Stack) vs 큐(Queue)

- **스택 (LIFO - Last In First Out):** 후입선출. 가장 나중에 들어온 것이 가장 먼저 나갑니다. (실행 컨텍스트가 작동하는 방식)
    
- **큐 (FIFO - First In First Out):** 선입선출. 줄 서기와 같이 먼저 들어온 것이 먼저 나갑니다.
    

### 실행 흐름 예시

1. **전역 컨텍스트:** 파일이 실행되는 순간 가장 먼저 쌓입니다.
    
2. **함수 실행:** 함수가 호출되면 해당 함수의 컨텍스트가 전역 컨텍스트 위에 쌓입니다.
    
3. **실행과 소멸:** 스택의 맨 위에 있는 컨텍스트가 활성화되며 코드를 실행합니다. 실행이 끝나면 스택에서 제거(Pop)되고, 바로 아래에 있던 컨텍스트가 다시 활성화됩니다.
    

> **Key Point:** "가장 최근에 들어온(위에 있는) 컨텍스트를 끝내야 그 아래의 명령어가 실행될 수 있다"는 점이 핵심입니다.

## 3. 실행 컨텍스트의 내부 구조

실행 컨텍스트 객체는 활성화 시점에 크게 3가지 정보를 수집합니다.

1. **VariableEnvironment (VE):** 현재 컨텍스트 내의 식별자 정보(변수명 등)와 외부 환경 정보를 담습니다. 선언 시점의 **스냅샷**으로, 변경 사항은 반영되지 않습니다.
    
2. **LexicalEnvironment (LE):** 초기에는 VE와 같지만, 코드 실행 도중 **변경 사항이 실시간으로 반영**됩니다. (주로 우리가 참고하는 데이터)
    
3. **ThisBinding:** `this` 식별자가 바라봐야 할 대상 객체를 저장합니다.
    

---

## 4. VariableEnvironment (VE) 자세히 보기

VariableEnvironment는 컨텍스트가 생성될 때의 상태를 그대로 유지하는 것이 특징입니다.

- **구성 요소:**
    
    - **environmentRecord:** 현재 컨텍스트 내부의 식별자 정보 (함수 그 자체, 변수명 등)
        
    - **outerEnvironmentReference:** 외부 환경을 참조하는 정보 (스코프 체인의 근거)
        
- **작동 방식:** 컨텍스트 생성 시점에 VE에 정보를 먼저 담고, 이를 그대로 복사해서 LexicalEnvironment를 만듭니다. 이후 코드 진행에 따른 변화는 LE에만 적용됩니다.
