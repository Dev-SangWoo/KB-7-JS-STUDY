undefined: 자바스크립트 엔진이 자동으로 부여하는 값
undefined는 사용자가 명시적으로 지정할 수도 있지만, 자바스크립트 엔진이  응당 어떤 값이 있어야 하는데 실제로 지정되지 않았을 때 자동으로 부여합니다. 

다음 세 가지 상황이 대표적입니다.

- 값을 대입하지 않은 변수: 데이터 영역의 메모리 주소를 지정하지 않은 식별자에 접근할 때입니다.
- 존재하지 않는 프로퍼티: 객체 내부의 존재하지 않는 프로퍼티에 접근하려고 할 때입니다.
- 함수의 반환값 부재: return 문이 없거나 호출되지 않는 함수의 실행 결과로 반환됩니다.
1. 배열에서의 undefined와 '비어있음(empty)의 차이

배열에서도 undefined 가 나타나지만 비어있는 요소 와는 명확히 구분됩니다.

비어있는 요소 :배열의 크기를 강제로 지정하거나  값을 건너뛰고 할당하면 발생합니다. 이는 인덱스라는  키값 자체가 존재하지 않음을 의미하며 forEach, map 등 배열의  순회 메서드 대상에서 제외됩니다.
직접 할당한 undefined: 이는 비록 비어있음을 의미할지라도 하나의 실존하는 데이터(값)로 취급됩니다. 따라서 배열의 순회 메서드가 이를 데이터로 인식하고 정상적으로 동작합니다.

null 사용자가 명시적으로 없음을 표현하는 값

자바스크립트 엔진이 반환하는 undefined와 달리 null은 사용자가 명시적으로 값이 없음을 나타내기 위해 고안된 데이터 타입입니다.

사용 권장:  본래의 의미에 따라 개발자가 의도적으로 비어있음을 표현할 때는 undefined 대신 null을 사용하는 것이 관례 입니다. 이렇게 하면 undefined가 떴을 때 그것이 엔진에 의한 것인지 개발자의 의도인지 명확히 구분할 수 있습니다.
주의사항: typeof null은  object를 반환하는데 이는 자바스크립트 자체의 유명한  버그 입니다. 따라서 어떤 변수가 null인지 확인할 때는 typeof를 사용해서는 안 됩니다.

1. undefined와 null 비교 및 판별법

두 값은 의미상 비슷해 보이지만 비교 연산 시 차이가 발생합니다.

- 동등 연산자(==): null == undefined는 true를 반환하여 두 값을 같다고 판단합니다. 따라서 이 연산자로는 두 값을 정확히 구분할 수 없습니다.
일치 연산자(===): null === undefined는 false를 반환합니다. 두 값의 타입이 엄밀히 다르기 때문입니다.
결론: 어떤 변수의 값이 null 인지 혹은 undefined 인지 정확히 판별하기 위해서는 반드시 일치 연산자( === )를 사용해야 합니다

정리

1. 자바스크립트 데이터 타입의 종류

자바스크립트의 데이터 타입은 크게 두 가지로 분류됩니다.

- 기본형 (Primitive type): 숫자(Number), 문자열(String), 불리언(Boolean), null, undefined, 심볼(Symbol) 등이 있으며, 값이 담긴 주솟값을 바로 복제합니다.
- 참조형 (Reference type): 객체(Object)를 중심으로 배열(Array), 함수(Function), 날짜(Date), 정규표현식(RegExp) 등이 포함되며, 값이 담긴 주솟값들로 이루어진 묶음을 가리키는 주솟값을 복제합니다.
1. 메모리와 데이터 할당의 원리

자바스크립트는 효율적인 데이터 관리를 위해 변수 영역과 데이터 영역을 분리하여 사용합니다.,

- 변수(Variable)와 식별자(Identifier): '변수'는 변할 수 있는 데이터가 담길 수 있는 공간을 '식별자'는 그 변수의 이름(변수명)을 의미합니다.
- 할당 과정: 변수를 선언하면 변수 영역에 공간을 확보하고 식별자를 저장합니다. 데이터를 할당할 때는 데이터 영역의 빈 공간에 실제 값을 저장하고 그 주소를 변수 영역의 값으로 대입합니다.
- 분리 이유: 동일한 데이터를 여러 변수에 할당할 때 중복 저장을 피하고 메모리를 효율적으로 관리하기 위함입니다.
1. 기본형의 불변성(Immutability)과 참조형의 가변성(Mutability)
- 기본형(불변값): 한 번 데이터 영역에 만들어진 기본형 데이터는 변경할 수 없습니다. 값을 바꾸려 하면 기존 주소의 값을 바꾸는 것이 아니라 새로운 값을 만들어 새로운 주소를 변수에 연결합니다.
- 참조형(가변값): 참조형 데이터는 내부의 프로퍼티(변수) 영역이 별도로 존재합니다. 객체의 프로퍼티를 변경하더라도 **객체 자체가 가리키는 주소는 변하지 않고 내부 프로퍼티의 주소만 바뀌기 때문에 가변적이라고 합니다.,
1. 불변 객체와 복사 (얕은 복사 vs 깊은 복사)

참조형 데이터의 가변성으로 인해 발생하는 부수 효과를 막기 위해 불변 객체 개념이 중요합니다.,

- 얕은 복사(Shallow Copy): 바로 아래 단계의 값만 복사하는 방식입니다. 중첩된 객체의 경우 주소만 복사되어 원본과 사본이 동일한 내부 객체를 공유하게 됩니다.,
- 깊은 복사(Deep Copy): 객체 내부의 모든 값을 재귀적으로 찾아 전부 복사하는 방식입니다. 이를 통해 원본과 완전히 독립된 새로운 객체를 만들 수 있으며 `JSON.parse(JSON.stringify(obj))` 등을 활용해 구현하기도 합니다.
1. undefined와 null의 차이

두 값 모두 '없음'을 나타내지만 명확한 차이가 있습니다.

- undefined: 자바스크립트 엔진이 자동으로 부여하는 값입니다. 값을 대입하지 않은 변수 존재하지 않는 프로퍼티 접근 return 문이 없는 함수의 결과 등에서 발생합니다.
- null:사용자가 명시적으로 '값이 없음'을 표현**하기 위해 사용하는 값입니다.,
- 주의사항:`typeof null`은 'object'를 반환하는 자바스크립트 자체 버그가 있으므로, 값을 판별할 때는 반드시 일치 연산자(`===`)를 사용해야 합니다
