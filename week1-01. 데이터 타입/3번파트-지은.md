# 01. 데이터 타입

## 내가 맡은 파트 : 2번

`1-3 변수 선언과 데이터 할당`

## 내용

## 1-3. 변수 선언과 데이터 할당 🧠

### 1. 변수 선언의 내부 원리 🏗️

변수 선언은 단순히 이름을 짓는 행위를 넘어, 메모리에 **데이터를 담을 공간을 확보하고 식별자를 연결하는 과정**이다.

#### [`var a`]

- **동작 방식:** 메모리에서 비어있는 공간(예: 주소 `1003`)을 확보한다.
- **식별자 연결:** 확보한 공간에 이름(예: `a`)을 붙인다.
- **상태:** 값을 할당하기 전까지는 빈 공간이거나 `undefined`가 담긴 그릇 상태이다.

---

### 2. 데이터 할당 방식: 영역의 분리 📍

자바스크립트는 효율적인 관리를 위해 **변수 영역**과 **데이터 영역**을 나누어 관리한다.

#### [` a = 'abc'`]

1. **변수 영역**의 빈 공간(@1003)을 확보하고 이름(`a`)을 지정한다.
2. **데이터 영역**의 빈 공간(@5004)에 실제 데이터 `'abc'`를 저장한다.
3. 변수 영역의 값 칸에 데이터 영역의 주소(`@5004`)를 저장한다.

---

### 3. 왜 영역을 나누어 저장할까? 🤔

데이터를 변수에 직접 저장하지 않고 주소를 참조하는 방식에는 기술적인 이유가 있다.

#### 3.1 효율적인 데이터 변환 (가변적 크기 대응) 📏

- 숫자형은 8바이트로 고정되지만, 문자열은 길이에 따라 필요한 용량이 계속 변한다.
- 만약 변수 영역에 데이터를 직접 저장하면, 데이터가 커질 때마다 뒤쪽 주소에 저장된 데이터를 모두 밀어내야 하는 비효율적인 연산이 발생한다.
- 분리 저장하면 데이터 영역에서 새로운 공간을 확보해 저장하고, 주소값만 업데이트하면 끝이다. ⚡

#### 3.2 중복 데이터 처리 효율성 💾

- 동일한 데이터(예: 숫자 `5`)를 500개의 변수에 할당하는 상황을 가정해보자.
  - **직접 저장:** 500개 공간마다 8바이트씩 차지 (총 4000바이트).
  - **분리 저장:** 데이터 영역에 `5`를 딱 한 번 저장하고 그 주소만 공유 (메모리 대폭 절약).

---

### 4. 데이터의 불변성 (Immutability) ❄️

이미 생성된 데이터 영역의 값은 직접 수정되지 않는다.

- **예시:** `'abc'`에 `'def'`를 추가해 `'abcdef'`를 만드는 경우
  - 기존 주소(@5004)의 데이터를 수정하는 것이 아니다.
  - 데이터 영역에 **새로운 문자열 `'abcdef'`를 생성**(@5005)한다.
  - 변수 `a`는 이제 새로운 데이터의 주소(@5005)를 가리킨다.
- **핵심:** 자바스크립트에서 데이터의 변화란, 기존 것을 바꾸는 게 아니라 **새로운 주소로 갈아 끼우는 것**이다.

---

## 핵심 키워드

#### 1. 변수 영역과 데이터 영역의 분리 🧱

- **정의:** 식별자가 담기는 공간(변수 영역)과 실제 데이터가 담기는 공간(데이터 영역)을 별도로 운영하는 방식이다.
- **이유:** 데이터의 크기는 가변적이기 때문에, 메모리 확보를 유연하게 하고 연산 효율을 높이기 위해 나누어 관리한다.

#### 2. 메모리 주소(Address) 참조 📍

- **정의:** 변수 영역에 데이터 자체를 직접 저장하는 것이 아니라, 데이터가 담긴 **메모리의 위치(주소값)**를 저장하는 방식이다.
- **장점:** 중복된 데이터를 여러 변수에 할당할 때 동일한 주소만 가리키면 되므로 메모리 낭비를 획기적으로 줄여준다.

#### 3. 데이터의 불변성 (Immutability) ❄️

- **정의:** 기본형 데이터(문자열, 숫자 등)는 한 번 메모리에 생성되면 **그 값이 직접 변경되지 않는다**는 성질이다.
- **동작:** 변수의 값을 변경하면 기존 주소의 데이터가 바뀌는 것이 아니라, **새로운 데이터를 만들어 그 주소를 변수에 다시 연결**한다.

#### 4. 가비지 컬렉터 (Garbage Collector, GC) 🧹

- **정의:** 데이터 영역에 저장되었으나, 어떤 변수에서도 더 이상 참조하지 않게 된(연결이 끊긴) 데이터를 자동으로 찾아 메모리에서 삭제해주는 역할이다.
- **의미:** 개발자가 수동으로 메모리를 관리하지 않아도 효율적인 상태를 유지해준다.

#### 5. 가변적 데이터 변환의 최적화 ⚡

- **정의:** 문자열처럼 길이가 변하기 쉬운 데이터를 처리할 때의 최적화 기법이다.
- **동작:** 분리 저장 방식 덕분에 데이터가 길어지거나 짧아져도 주변 메모리 데이터를 밀어내거나 당길 필요 없이, 새로운 공간만 확보하면 되므로 처리 속도가 빠르다.
