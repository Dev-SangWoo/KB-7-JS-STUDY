# JS 데이터 타입

## 데이터 타입 종류

JS의 데이터 타입은 2가지

| 구분 | 종류 |
|------|------|
| 기본형 (불변형) | 숫자, 문자열, 불리언, null, undefined |
| 참조형 (가변형) | 객체, 배열, 함수, 날짜 등 |

> 모든 데이터는 메모리 주소값을 통해 서로 구분하고 연결 가능

---

## 기본형 데이터 선언 동작 원리

- 변수를 선언할 때 메모리 영역 어딘가에 변수명과 값의 주소가 등록되고, 값은 또 다른 주소에 입력된다
- 값이 바뀌면 기존 데이터 주소를 건들지 않고 새로운 주소를 할당시키는 동시에 주소값만 변경된다

**장점** : 데이터와 변수명을 따로 저장하기 때문에 데이터가 같으면 메모리를 낭비하지 않고 똑같은 주소를 넘겨준다

---

## 참조형 데이터 선언 동작 원리

- 참조형 데이터는 3층 구조로 저장된다
- 데이터가 변경될 때 맨 처음 데이터를 가리키는 주소값은 변하지 않고, 내부 값은 얼마든지 바꿀 수 있다

---

## 불변 객체 / 얕은 복사 / 깊은 복사

| 개념 | 설명 |
|------|------|
| 불변 객체 | 전달받은 객체에 변경을 주더라도 원본 객체에는 변화를 주지 않은 것 |
| 얕은 복사 | 1단계 프로퍼티만 복사. 기본형은 독립, 내부에 참조형이 있으면 주소값만 복사해서 원본과 공유 |
| 깊은 복사 | 내부 참조형까지 반복해서 복사. 모든 단계의 주소값이 새로 생성되어 원본과 완전 독립 |

## undefined

다음 경우에 자동으로 할당된다

- 값을 대입하지 않았을 때
- 객체 내부에 존재하지 않는 프로퍼티에 접근하려 할 때
- return문이 없을 때
