# 📚 코어 자바스크립트 5주 딥다이브 스터디

프론트엔드 기술 면접의 바이블, 『코어 자바스크립트』를 5주 만에 완벽하게 내 것으로 만드는 스터디입니다. 단순 완독을 넘어, 핵심 개념을 '내 언어로 명확하게 설명'할 수 있는 수준까지 끌어올리는 것을 목표로 합니다.

## 🎯 스터디 개요

- **진행 기간:** 2026.04.01 ~ 2026.05.06 (총 5주)
- **모임 횟수:** 주 1회 (총 5회)
- **모임 요일:** 매주 수요일 / 오후 6시 / 5층 회의실
- **참여 인원:** 6명

## 🏗️ 아키텍처 및 사용 기술

- **Language:** JavaScript (ES6+)
- **Version Control:** Git & GitHub
- **Communication:** In-person

## 📌 스터디 진행 규칙

1. **전체 범위 정독 필수:** 매주 정해진 진도 분량은 스터디원 전원이 끝까지 읽어옵니다.
2. **파트 분배 발제:** 모임 전, 해당 주차의 세부 목차를 1/N로 나누어 각자 맡은 파트만 깊게 파고들어 요약 자료를 준비합니다.
3. **5분 릴레이 발표:** 모임 당일, 1인당 딱 **5분씩** 본인 파트의 핵심과 면접 예상 질문 위주로 임팩트 있게 발표합니다.
4. **아카이빙:** 발표 자료는 본 레포지토리에 PR(Pull Request) 방식으로 업로드하여 포트폴리오로 활용합니다.

## 🛠️ 스터디 레포지토리 컨벤션 (Convention)

`이슈 / 브랜치 / 커밋/ PR `

### 1. 이슈(Issue) 네이밍 규칙

> - 매주 메인 이슈 생성 예정
> - 해당 메인 이슈에 연결

형식 : [week주차] - 범위 제목 - 내 범위

예시 :

`제목 : [week1] - 01장. 데이터 타입 - 1-1데이터타입의종류`

`내용 : #1 (메인 이슈 링크 걸기)`

### 2. 브랜치(Branch) 네이밍 규칙

각자 자신의 브랜치를 생성하여 작업한 후 main 브랜치로 PR을 날립니다.

형식: `week주차/Issue-나의 이슈번호/이름`

예시: `week1/Issue-1/sangwoo`

### 3. 커밋(Commit) 메시지 규칙

통일된 커밋 메시지를 위해 [Gitmoji]와 태그를 활용합니다.

형식: 태그: 작업 내용 요약

태그 종류:

- `📝 docs: 마크다운 요약 문서 추가 및 수정 (가장 많이 쓰임)`
- `🐛 fix: 문서 오타, 잘못된 정보, 에러 수정`

예시:

```
docs: 1주차 3번 파트(기본형과 참조형) 요약 정리

fix: 실행 컨텍스트 파트 오타 수정
```

### 4. PR (Pull Request) 규칙

PR 제목만 봐도 누가 어떤 주차의 과제를 냈는지 한눈에 알 수 있도록 통일합니다.

PR 제목 형식:`week주차/Issue-나의 이슈번호/이름`

PR 제목 예시: `week1/Issue-1/sangwoo`

### 5. 머지(Merge) 및 코드 리뷰 룰

PR을 올리면 다른 팀원들이 코멘트(리뷰)를 남겨줍니다.

발표 모임이 끝난 후, 궁금증이 다 해결되면 팀원 중 1명 이상의 Approve를 받은 PR만 main 브랜치로 머지합니다.

충돌(Conflict)이 발생할 경우 PR 작성자가 직접 해결한 후 머지합니다.
